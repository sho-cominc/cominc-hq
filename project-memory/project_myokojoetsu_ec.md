# myokojoetsu.com お土産EC — チームブリーフィング

2026-05-01 / Cat・Law・Fin・Clare・Yuri・Webber 合同リサーチ

---

## 0. 概要

myokojoetsu.com（妙高・上越の地域メディアサイト）に**お土産の物販機能**を追加する。
品数は少なく、本当に良いと思うものだけ。「ただの通販」ではなく、**メディア（記事）× コマース（物販）× ツーリズム（体験）の三位一体**で勝負する。

## 1. Cat（マーケティング）: 成功事例と戦略

### ロールモデル 3 選

**うなぎの寝床**（福岡・八女）— 最も直接的な手本
- 自称「地域文化商社」。10年で年商30倍（約2.4億円）、赤字なし
- 久留米絣のもんぺ: 年間2万着。「情報を売るものづくり」
- サイクル: リサーチ → 取材 → 編集 → 販売 → ツーリズム
- Sho との共通点: 地域に住み、作り手と直接会え、リアルの場（古民家）を持つ

**THE**（中川淳 × 水野学 × 鈴木啓太）
- 「各カテゴリで THE と呼べるモノを1つだけ選ぶ」
- 5-7品でスタートしても全然いい。「たった7品しかない」が最強の差別化

**Food52 / Provisions**（NY）
- レシピメディア → キュレーション物販 → $100M メディア帝国
- 商品ページではなく**記事にリンク** → 読者エンゲージメント → 購入率向上

## 2. Law（法務）: やるべきこと・ブロッカー

### 赤フラグ（対応必須・時間かかる）

| # | 項目 | 内容 | 期間 | 費用 |
|---|------|------|------|------|
| 1 | **通信販売酒類小売業免許** | 日本酒を売るなら必須。上越税務署に申請 | 2ヶ月〜 | 3万円 |
| 2 | **食品衛生責任者** | 1日講習で取得。新潟県食品衛生協会（eラーニングも可） | 1日 | 約1万円 |
| 3 | **特商法に基づく表記ページ** | 事業者名・住所・電話・返品条件など。サイトに必須 | 1日 | 0円 |

### 酒類免許の重要ポイント

- 国産酒は「課税移出数量が全品目3,000kl未満の製造者」の製品に限定
- 妙高・上越の地酒蔵（小規模）は大半が対象内と推定
- 蔵元から**課税移出数量証明書**を取得する必要あり → 蔵元との事前交渉
- サイトに**年齢確認の入力欄**（生年月日入力方式）が必要
- 申請にはサイトのデモ画面（年齢確認ページ含む）の提出が求められる

### 海外販売の法的壁

- 海外への酒類販売は**輸出酒類卸売業免許**が別途必要
- EMSでアメリカ・欧州主要国への酒類発送は**不可**
- **結論: 海外向け酒類販売は初期は見送り。海外は工芸品・乾燥食品のみ**

### 食品販売の分類

| 商品 | 必要な手続き |
|------|------------|
| 包装済み常温食品（米・乾麺・瓶詰め）をそのまま販売 | 届出不要 or 営業届出のみ |
| 漬物（仕入れ品そのまま） | 営業届出 |
| 漬物（自家製） | 漬物製造業許可（2021年新設） |
| 陶器・工芸品 | 不要 |
| 日本酒 | 通信販売酒類小売業免許 |

### 個人事業主でOK

法人化は不要。全許認可が個人事業主で取得可能。法人化検討は課税所得800万超になってから。

### 推奨アクション順

1. 食品衛生責任者講習の受講（1日・1万円）
2. 上越保健所に相談（取扱商品の許可/届出の要否確認）
3. 通信販売酒類小売業免許の申請（上越税務署・2ヶ月）
4. 蔵元に課税移出数量証明書を依頼
5. 特商法の表記ページ作成
6. プラットフォーム選定

- 手数料15-20%が Sho の収入
- ComInc. の「Cultural Translation」ミッションとの整合性が最高

## 4. Clare（クリエイティブ）: デザイン方針

### コンセプト: 「雑誌の中にBuyボタンがある」

典型的な EC グリッド（商品カード4列×N行）には**しない**。
myokojoetsu.com の editorial トーン（beige + serif italic + 余白）を維持し、記事の中に購入導線を溶け込ませる。

### 商品ページのレイアウト

```
┌─────────────────────────────────┐
│ [Full-bleed 写真: 作り手のポートレート]  │
│                                         │
│   かんずり                              │
│   — 雪の上で唐辛子を晒す、              │
│     妙高の冬の風物詩                     │
│                                         │
├─────────────────────────────────┤
│                                         │
│  [記事本文: 1500字]                      │
│  作り手の話、工程、歴史、使い方          │
│  途中に写真3-4枚                         │
│                                         │
├─────────────────────────────────┤
│                                         │
│  ┌──────────────┐                       │
│  │ かんずり 70g   │                     │
│  │ ¥864（税込）   │                     │
│  │ [購入する →]   │ ← STORES に遷移    │
│  └──────────────┘                       │
│                                         │
└─────────────────────────────────┘
```

### ビジュアル参考

- **Food52 Provisions**: 記事に自然に溶け込む商品紹介
- **D&DEPARTMENT**: 棚がそのままメディアになっている
- **うなぎの寝床**: もんぺの1点突破。深い記事+商品リンク

### 写真ディレクション

| カット | 内容 | トーン |
|--------|------|--------|
| メイン | 商品単体。自然光、白 or cream 背景 | 静謐・上品 |
| 作り手 | ポートレート。作業場で。笑顔より集中の表情 | ドキュメンタリー |
| 工程 | 制作過程の瞬間（雪上のかんずり、蒸米 etc） | 臨場感 |
| 使用シーン | 料理に使っている、酒を注いでいる | 日常の美 |

### やらないこと

- カート / 商品一覧グリッド → STORES 側に任せる
- セール / 割引バッジ → プレミアム感を損なう
- 星評価 / レビュー → 最初は不要。信頼はストーリーで獲得

## 5. Yuri（UX）: ユーザーフロー

### 導線設計

```
[myokojoetsu.com 記事を読む]
        ↓
[記事末尾の「購入する」ボタン]
        ↓
[STORES の商品ページへ遷移]
        ↓
[STORES 標準カート → 決済 → 完了]
```

### 体験→購入パイプライン（Sho 独自）

```
[古民家に宿泊 / ツアーに参加]
        ↓
[食事でかんずりを体験 / 酒蔵を見学]
        ↓
[チェックアウト時に QR コード付きカードを渡す]
        ↓
[帰宅後 myokojoetsu.com で記事を読む]
        ↓
[「あの味をもう一度」→ 購入]
```

### UX 原則

1. **3クリックルール**: 記事 → 購入ボタン → STORES 決済。最大3ステップ
2. **記事ファースト**: 商品スペック表ではなくストーリーを先に見せる
3. **Mobile-first**: 旅行者はスマホで再訪する
4. **言語**: 英語メイン + 日本語併記（固有名詞・雰囲気）
5. **カートは STORES に委託**: 自前で作らない

## 6. Webber（Web ディレクター）: サイト構成

### アーキテクチャ

```
myokojoetsu.com（静的 HTML / Cloudflare Pages）
├── index.html          ← 地域メディア LP
├── /shop/              ← 商品記事の一覧（将来）
│   ├── kanzuri.html    ← 記事ページ（1商品=1記事）
│   ├── ayumasamune.html
│   └── ...
└── → 購入ボタンは STORES のリンクに飛ばす

STORES（または Shopify）
├── 商品登録（5-7品）
├── カート・決済・配送管理
└── 特商法表記ページ
```

### なぜデカップルド（分離型）か

- myokojoetsu.com はメディア。STORES はコマース。役割を分ける
- メディア側は自由にデザインできる（STORES のテンプレに縛られない）
- STORES は決済・物流・法的テンプレを全部やってくれる
- 将来 Shopify に切り替えてもメディア側は変更不要

### 初期実装（Phase 1）

- myokojoetsu.com LP に「Shop」セクションを追加（カテゴリの1つとして）
- 各商品のエディトリアル記事ページはまだ作らない（LP 内で軽く紹介）
- 「購入はこちら →」で STORES の商品ページへ外部リンク

### Phase 2 以降

- 各商品の記事ページを個別 HTML で作成
- Shopify Buy Button の埋め込み（ページ内で決済完結）
- ブログ / ジャーナル機能（Astro + Notion CMS 移行時に）

## 8. Sho の最大の武器（まとめ）

Amazon にも楽天にも新潟直送計画にも**ない**もの:

1. **バイリンガル**: 英語でストーリーを語れる → 越境ECのブルーオーシャン
2. **現地在住**: 作り手と直接会い、「深い」ストーリーを書ける
3. **ツアー事業との連動**: 体験→購入パイプラインは唯一無二
4. **古民家という舞台**: ゲストに商品を体感させる場がある
5. **メディアサイトが既にある**: コンテンツの器は準備済み

**結論**: 「ただの通販」にしない。**メディア × コマース × ツーリズムの三位一体**。
